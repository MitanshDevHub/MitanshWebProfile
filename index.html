SECURITY DEVICE SIMULATION

Course: CS 330: Discrete Structures - IIT Chicago

Synopsis: This program simulates a security device akin to those used in homes and offices. Users or random number generators input strings, with the program deciding if the correct access code has been entered. The program provides feedback on its current state (locked or unlocked) and furnishes statistics about the number of attempts, successful unlocks, and locks.

Key Features:

  • SecurityEngine.py: Manages the locking and unlocking mechanism. Accepts user input to alter the device's state, filters out non-digit characters, and keeps stats on the number of attempts and the state changes.
  • ranNumGen.py: Simulates brute force attacks on the security device. This class attempts to unlock the device without any time delay and then computes the average time required if there's a one-second wait between each attempt.

Findings: While a human might take roughly 7-8 days to break the lock given a one-second wait time, computers can decipher the code in mere seconds. It accentuates the importance of robust cybersecurity in this digital age.

Conclusion: With the advent of quantum computing, it's paramount to re-evaluate and fortify security measures. This project was implemented using Python and is grounded on the principles of Finite Automata.

REAL ESTATE MANAGEMENT APPLICATION

Course: CS-425: Database Organization - IIT Chicago

Synopsis:
This application is a conceptual embodiment of a comprehensive real estate management solution. Developed using Django in conjunction with HTML/CSS/JavaScript, it's designed to proficiently handle property listings, streamline the relationships between clients, landlords, and agents, and oversee transactions in the real estate sector. Although the underlying code is robust and feature-rich, there were challenges in achieving seamless front-end to back-end communication, making it more of a representational prototype than a fully operational platform.

Achievements & Features:

  • Tech Stack: Utilized Django as the back-end framework combined with HTML/CSS/JavaScript for front-end development, forming the base of the application.
  • Database Design: Architected and rolled out a robust PostgreSQL database schema which is the backbone for all the functionalities provided by the application, from listing properties to managing transactions.
  • ER to Relational Mapping: Initiated the process with an ER-model and then adeptly translated it into a functional relational schema using SQL scripts.
  • Challenges Met: Encountered challenges like form validation discrepancies, unexpected server responses, and intricacies of Django's template system. While I addressed many of these concerns, some complexities, especially in the front-end to back-end communication,remained.
  • Integrated Solution: Aimed to ensure a harmonious blend of the front-end and back-end components for an ideal user experience. However, some integration challenges persisted, emphasizing the prototype nature of the project.
